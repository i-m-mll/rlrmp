# anti_anticipation_smoke

Local CPU smoke runs validating the `EpochMaskedLoss` wiring on
`feature/anti-anticipation-loss` (rlrmp side) + `feature/loss-epoch-masked`
(feedbax side, issue `50507a9`). Each run is `--n-warmup-batches 5
--batch-size 50 --n-replicates 2 --no-streaming-loss`.

| Variant                       | New flag(s)                                                    | Purpose                                       |
|-------------------------------|----------------------------------------------------------------|-----------------------------------------------|
| `baseline_gru__none_repro`    | (none)                                                         | regression check vs prior baseline            |
| `baseline_gru__motor_pre_go`  | `--nn-output-pre-go 1e-2`                                       | wraps `nn_output` in `EpochMaskedLoss`        |
| `baseline_gru__hidden_pre_go` | `--nn-hidden-derivative-pre-go 1e-2`                            | wraps `StateDerivativeLoss` in `EpochMaskedLoss` |
| `baseline_gru__jerk_motor_pre_go` | `--nn-output-jerk 1e5 --nn-output-pre-go 1e-2`                 | layered intervention (proposed primary)       |

Heavy artifacts (`.eqx`, `.npz`) under
`_artifacts/anti_anticipation_smoke/<variant>/` (gitignored). The
`run.json` files in this tree are auto-generated by `train_minimax.py`
and serve as the run-spec for each smoke variant.

Tracking: `efc4d68` (rlrmp) + `50507a9` (feedbax).
